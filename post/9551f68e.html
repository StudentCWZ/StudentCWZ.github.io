分支进阶与版本回退

Fast-Forward

  1. 如果可能，合并分支时 Git 会使用 fast-forward 模式。
  2. 在这种模式下，删除分支时会丢掉分支信息
  3. 合并时加上 –no-ff 参数会禁用 fast-forward ，这样会多出一个 commit id
  4. 查看 log

版本回退

  1. 把当前文件目录下的所有文件放入暂存区，然后提交
  2. 回退到上一版本
  3. 返回到某一版本
